Language Guide

​

Learn the core concepts of WebFluent.

​

Pages

​

Pages are top-level route targets. Each page defines a URL path and contains the UI tree for that route.

​
Page Home (path: "/", title: "Home") { Container { Heading("Welcome", h1) Text("This is the home page.") } }​
​
​

Page attributes:

​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​
AttributeTypeDescription
pathStringURL route for this page (required)
titleStringDocument title
guardExpressionNavigation guard — redirects if false
redirectStringRedirect target when guard fails
​

Components

​

Reusable UI blocks that accept props and can have internal state.

​
Component UserCard (name: String, role: String, active: Bool = true) { Card(elevated) { Row(align: center, gap: md) { Avatar(initials: "U", primary) Stack { Text(name, bold) Text(role, muted) } if active { Badge("Active", success) } } } } // Usage UserCard(name: "Monzer", role: "Developer")​
​
​

Props support types: String, Number, Bool, List, Map. Optional props use ?, defaults use =.

​

State and Reactivity

​

State is declared with the state keyword. It is reactive — any UI that reads it updates automatically when it changes.

​
Page Counter (path: "/counter") { state count = 0 Container { Text("Count: {count}") Button("+1", primary) { count = count + 1 } Button("-1") { count = count - 1 } } }​
​
​

Derived state:

​
state items = [{name: "A", price: 3}, {name: "B", price: 2}] derived total = items.map(i => i.price).sum() derived isEmpty = items.length == 0​
​
​

Events

​

Event handlers are declared with on:event or via shorthand blocks on buttons.

​
Button("Submit") { on:click { submitForm() } } Input(text, placeholder: "Search...") { on:input { searchQuery = value } on:keydown { if key == "Enter" { performSearch() } } } // Shorthand: block on Button defaults to on:click Button("Save") { save() }​
​
​

Supported events: on:click, on:submit, on:input, on:change, on:focus, on:blur, on:keydown, on:keyup, on:mouseover, on:mouseout, on:mount, on:unmount

​

Control Flow

​

Conditionals:

​
if isLoggedIn { Text("Welcome back!") } else if isGuest { Text("Hello, guest") } else { Button("Log In") { navigate("/login") } }​
​
​

Loops:

​
for user in users { UserCard(name: user.name, role: user.role) } // With index for item, index in items { Text("{index + 1}. {item}") }​
​
​

Show/Hide (keeps element in DOM, toggles visibility):

​
show isExpanded { Card { Text("Expanded content") } }​
​
​

Stores

​

Stores hold shared state accessible from any page or component.

​
Store CartStore { state items = [] derived total = items.map(i => i.price * i.quantity).sum() derived count = items.length action addItem(product: Map) { items.push({ id: product.id, name: product.name, price: product.price, quantity: 1 }) } action removeItem(id: Number) { items = items.filter(i => i.id != id) } } // Usage in a page Page Cart (path: "/cart") { use CartStore Text("Total: ${CartStore.total}") Button("Clear") { CartStore.clear() } }​
​
​

Routing

​

SPA routing is declared in the App file.

​
App { Navbar { Navbar.Brand { Text("My App", heading) } Navbar.Links { Link(to: "/") { Text("Home") } Link(to: "/about") { Text("About") } } } Router { Route(path: "/", page: Home) Route(path: "/about", page: About) Route(path: "/user/:id", page: UserProfile) Route(path: "*", page: NotFound) } } // Programmatic navigation Button("Go Home") { navigate("/") } // Dynamic routes access params Page UserProfile (path: "/user/:id") { Text("User ID: {params.id}") }​
​
​

Data Fetching

​

Built-in async data loading with automatic loading, error, and success states.

​
fetch users from "/api/users" { loading { Spinner() } error (err) { Alert("Failed to load users", danger) } success { for user in users { UserCard(name: user.name, role: user.role) } } } // With options fetch result from "/api/submit" (method: "POST", body: { name: name, email: email }) { success { Alert("Saved!", success) } }​
