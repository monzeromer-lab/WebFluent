Internationalization (i18n)

​

Built-in multi-language support with reactive locale switching and automatic RTL.

​

Setup

​

Create a JSON file per locale in your translations directory.

​
// src/translations/en.json { "greeting": "Hello, {name}!", "nav.home": "Home", "nav.about": "About" } // src/translations/ar.json { "greeting": "!أهلاً، {name}", "nav.home": "الرئيسية", "nav.about": "حول" }​
​
​

Add i18n config to webfluent.app.json:

​
{ "i18n": { "defaultLocale": "en", "locales": ["en", "ar"], "dir": "src/translations" } }​
​
​

The t() Function

​

Use t() to look up translated text. It is reactive — all t() calls update when the locale changes.

​
// Simple key lookup Text(t("nav.home")) // With interpolation Text(t("greeting", name: user.name)) // In any component Button(t("actions.save"), primary) Heading(t("page.title"), h1)​
​
​

Locale Switching

​

Switch the locale at runtime with setLocale(). All translated text updates instantly.

​
Button("English") { setLocale("en") } Button("العربية") { setLocale("ar") } Button("Espanol") { setLocale("es") } // Access current locale Text("Current: {locale}") Text("Direction: {dir}")​
​
​

RTL Support

​

WebFluent automatically detects RTL locales and updates the document direction.

​
​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​
LocaleDirection
ar (Arabic)RTL
he (Hebrew)RTL
fa (Farsi)RTL
ur (Urdu)RTL
All othersLTR
​

When setLocale("ar") is called, the HTML element gets dir="rtl" and lang="ar" automatically.

​

Fallback Behavior

​

If a key is missing in the current locale:

​

1. Falls back to the defaultLocale translation

​

2. If still missing, returns the key itself (e.g., "nav.home")

​
​

SSG + i18n

​

When both SSG and i18n are enabled, pages are pre-rendered with the default locale text. After JavaScript loads, locale switching works normally.
