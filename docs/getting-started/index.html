Getting Started

​

Get up and running with WebFluent in under a minute.

​

Installation

​

Build from source. You need Rust installed.

​
git clone https://github.com/user/webfluent.git cd webfluent cargo build --release​
​
​

The binary is at target/release/webfluent. Add it to your PATH.

​

Create a Project

​

WebFluent comes with two starter templates.

​
SPA​

Interactive App

​

Dashboard with routing, stores, forms, modals, animations. Full CRUD task manager.

​
webfluent init my-app --template spa​
​
​
​
Static​

Static Site

​

Marketing site with SSG, i18n, blog cards, contact form. Pre-rendered HTML for instant loads.

​
webfluent init my-site --template static​
​
​
​
​

Build and Serve

​
cd my-app webfluent build webfluent serve​
​
​

Open http://localhost:3000 in your browser.

​

Project Structure

​

A WebFluent project has this structure:

​
my-app/ ├── webfluent.app.json # Project config ├── src/ │ ├── App.wf # Root app (router, layout) │ ├── pages/ # Page files │ │ ├── Home.wf │ │ └── About.wf │ ├── components/ # Reusable components │ │ └── Header.wf │ ├── stores/ # Shared state │ │ └── auth.wf │ └── translations/ # i18n JSON files │ ├── en.json │ └── ar.json ├── public/ # Static assets └── build/ # Compiled output​
​
​

Build Output

​

The compiler generates three files:

​
​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​ ​
FileContents
index.htmlHTML shell (or pre-rendered pages with SSG)
app.jsReactive runtime + compiled pages and components
styles.cssDesign tokens as CSS custom properties + component styles
